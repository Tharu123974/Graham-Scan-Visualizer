Graham Scan - Theory & Help

1. What is the Convex Hull?

The Convex Hull of a set of points is the smallest convex polygon that contains all the points of it. Imagine stretching a rubber band around the outermost points in a wooden pegboard; the boundary the band forms when it snaps tight is precisely the convex hull.

A polygon is defined as "convex" if the line segment connecting any two points inside the polygon lies entirely inside the polygon. Therefore, computing the convex hull creates the absolute minimal bounding convex shape encompassing the entire dataset.

2. Real Time Applications

  • Collision Detection: In video games, physics simulations, and robotics, determining if complex 3D or 2D objects interect is computationally expensive. Substituting a complex mesh with its Convex Hull allows engines to discard collision checks instantly if the hulls don't intersect.
  • Computer Vision & Pattern Recognition: Used to track finger movements in hand-gesture recognition and mapping boundaries of organ regions in MRI scans.
  • Geographic Information Systems (GIS): Defining the territorial ranges of migratory animal coordinates, analyzing watershed boundaries, or evaluating epidemiological disease outbreak clusters.
  • Mathematics & Logistics: Solving the farthest-pair problem computationally efficiently since the farthest two points in a dataset will eternally lie squarely on the perimeter of its convex hull.

3. Procedure (Graham Scan Algorithm)

The Graham's Scan method solves the convex hull linearly in optimal time by deploying an angle-sorting mechanism and a stack data structure.

  1. Find the Pivot Point: Identify the point P0 having the lowest Y coordinate. If there is a tie, select the point with the lowest X coordinate.
  2. Sort by Polar Angle: Sort all remaining points (P1, P2, ... Pn) by their polar angle in counterclockwise order with respect to the pivot P0.
  3. Initialize Stack: Push P0 and P1 onto the computational stack.
  4. Iterate & Screen: For every subsequent point Pi:
    • Compute the geometry between the top two points on the stack and the new point Pi.
    • If the trio traces a Right Turn or constitutes a collinear flat-line, POP the pinnacle origin point off from the stack and re-assess.
  5. Push Sequence: When the computational trajectory forms a Left Turn (Counter-Clockwise), push Pi strictly onto the stack. Move iteratively until all points are processed. The finalized stack reveals your Convex Hull boundary!

4. The Determinant Formula (Orientation Mapping)

To mathematically determine whether evaluating three consecutive coordinates traces a Counter-Clockwise (Left), Clockwise (Right), or Collinear (Straight) line, the Graham Scan harnesses a 2D cross-product evaluation rooted essentially on a 3 × 3 matrix determinant spanning coordinates A(x1, y1), B(x2, y2), and C(x3, y3):

x1 y1 1
x2 y2 1
x3 y3 1

Expanded mathematically into offline scalar mapping without requiring external matrix libraries, the computationally optimized expression evaluates:

Cross Product Formula (Δ) = (x2 - x1)(y3 - y1) - (y2 - y1)(x3 - x1)

We interpret the resultant value Δ natively as:

  • If Δ > 0 : Counter-Clockwise (Left Turn). Polygon boundary is confirmed dynamically valid.
  • If Δ < 0 : Clockwise (Right Turn). Internal concavity is detected (Pop the stack!).
  • If Δ = 0 : Collinear alignment. Erase mid-point anomalies and proceed to outermost apex.

5. Time & Space Complexity

Time Complexity

O(N log N)

Finding the fundamental pivot P0 takes O(N). Screening standard cross-products operates identically in linear O(N) stack phases. However, executing a complete Polar Angle Sort dominates at O(N log N) bounds.

Space Complexity

O(N)

In worst-case scenarios—where every point evaluates sequentially to the hull perimeters independently (e.g. a circular perimeter configuration)—the dynamic Stack structure will occupy O(N) proportional coordinate space directly mapping linearly.
